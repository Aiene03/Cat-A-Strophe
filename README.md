Cat-A-Strophe
				REFLECTION

Cat-A-Strophe is a Frogger-inspired endless crosser that is developed in React Native and Expo in which the player is a cat with nine lives to tap to jump forward across procedurally generated lanes of roads, railways and water. The obstacles visible in each lane, cars, trains and hazards of the river, which the player has to avoid, have their own twist: you have nine available cats, each with a distinctive name, fur color, eye color, and collar color. Upon the death of a cat, it goes to be buried in a graveyard screen and the next cat comes in, and once all the nine cats are killed, the game is finished, though an idleness time-out provides a UFO abduction death with a full beam-up effect, which allows humor and tension to be felt. The visuals are dark and minimalistic black backgrounds, subtle grid overlays, monospace lettering, and the aim is to create a retro atmosphere, pick-up-and-play mobile game that is both approachable yet harsh. The obstacle freeze bug proved by far the most difficult obstacle as the game loop is based on requestAnimationFrame which updates the position of obstacles at every frame, at the same time tap inputs update the position of the cat. These two authors to the same React state were engaged in unceasing conflict with one another all the time - a tap would cause a state update and the next animation frame would read out stale state on a ref and write the results of the tap over the top, freezing obstacles visually. I struggled through a few options: functional state updaters, ref-as-source-of-truth, single flush per frame and meticulously ensuring the separation of the tap write path and the render path of the loop since getting both systems to run together required a thorough grasp of how React batches and how the requestAnimationFrame events and React render cycles interacted in timing. Another level, the death detection, was necessary, i.e. the stepForward could not cause a change of state (i.e. to a death) but rather the change of state had to be flushed instantly to cause the death animation, whereas with normal taps, it could wait until the next loop frame. Over time, I would put the game loop and the full JS thread entirely out of the JS thread with Reanimated worklets or a native module, doing away with the React state bottleneck completely, and would add lateral (up/down) swipe movement so the cat only has to leap over lanes, not just forward. The sound system is a simple procedural system, which functions, but I would replace it with more sophisticated multi-channel chiptune audio, possibly through a Web Audio-style system with filters and effects. I would also include visual flourishes: the particle effects on death, water tiles animation, beam of headlights on night cars, and a day/night cycle linked with the level. It would be finished off with a persistent leaderboard with online scoring, haptic feedback patterns on various surfaces, and an animated tutorial on first-time players, and lastly, the engine and collision logic would be unit-tested to make further iteration safer.
